# From ConfigMgr to Manager of Configs

## Abstract

What if managing servers wasn’t about fine-grained configuration, but about making sure any server could run what it needed, wherever it lived?

Making the jump from on-prem to the cloud often feels daunting. Legacy systems, GPOs that just won’t die, and large applications that deploy perfectly on-prem raise a fair question: why change?

In this session, we’ll look at how to shift that model using PowerShell, Azure Arc, and centralized artifact repositories. The focus is on managing servers like cattle, not pets. Configuration and intent travel with the workload, applications become portable, and management scales without chasing a 1:1 replacement for ConfigMgr or GPOs.

You’ll leave with a management model that scales, travels with your workloads, and stops infrastructure from being the thing that slows everything else down.
