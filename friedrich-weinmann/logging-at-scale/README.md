# Logging At Scale

## Abstract

A quick logfile is swiftly implemented, so we can go back on solving our actual problem that has us reaching for our script editor! But what if we need to go beyond the bare minimum? What if we need more useful information? More flexibility? Performance? Log from multiple runspaces/threads in parallel?

In fact, as our code becomes more complex, our requirements for logging grow along with it.

But ...

Logging is also never the problem we need to solve right now! It is always something we just don't have the time to Do Right. Not because we don't want it to be great, but because we rarely can afford to spend the time to make it so.

Especially when it would be the most convenient to have.

Introducing the PSFramework module's logging feature, that tries to solve all that ...
