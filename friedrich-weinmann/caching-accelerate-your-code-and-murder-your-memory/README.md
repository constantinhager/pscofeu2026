# Caching - Accelerate your Code and Murder your Memory

## Abstract

Working with live data is great, until it ain't.

Asking for the same data again and again can add up delays in our code, and lead us to performing redundant actions.

This is where caching comes in, to save the day: We store the data locally - in memory or on disk - and save loads of time.

Until things go wrong. Again.

So lets discuss strategies, techniques and tools to make it better. Deal with problems like stale data, memory load, parallel access or sharing a cache between commands responsibly.
