# Designing modular code

## Abstract

We’ve all seen those 1,000-line scripts that have slowly grown over the years. They run mission-critical tasks and represent hundreds of hours of work — yet no one dares to touch them anymore. Instead, everyone just hopes they’ll keep working somehow.
What if a few simple techniques could change that? Imagine your production scripts shrinking to just a handful of manageable lines — while still reusing all the effort already invested.
In this session, I’ll show a set of lightweight tools that help you break large scripts into modular, reusable parts. You’ll also see how to easily reassemble those parts into self-contained production scripts ready for deployment — no dependencies required.
