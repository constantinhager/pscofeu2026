# Think Before You Loop: "Big O Thinking" for PowerShell devs

## Abstract

Does this sound familiar: You have written a beautiful PowerShell script that runs smoothly with 100 objects. However, when you try to run it in a larger production environment, suddenly everything crashes?
In this scenario, most PowerShell developers reach for tools such as parallel processing, different cmdlets, or .NET calls to circumvent the issue. 
But what if the real problem is not in your tool stack, but in how you are thinking about the problem to begin with? What if 30 seconds of algorithmic thinking could eliminate hours of optimization work?

In this session, I will introduce Big O Thinking - not as an academic theory, but as the practical thinking tool it has become for me ever since learning about it in Computer Science. No complex mathematics. Just the mental framework that makes you ask, "how many loops will this code create?" before you write it. Essentially, Big O is the idea of thinking through what Jakub's profiler would tell you about your code.

After this session, you will have a new mental framework that prevents performance problems before they happen. Instead of writing first and profiling later, you will think algorithmically from the start. You will walk away with practical decision trees for common PowerShell scenarios and the confidence to choose the right approach for any scale. This is basic Computer Science theory, and it is not scary. It is the thinking tool that has guided my coding for over 20 years.
