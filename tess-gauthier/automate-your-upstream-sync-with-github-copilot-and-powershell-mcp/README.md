# Automate your upstream sync with GitHub Copilot and PowerShell MCP

## Abstract

OpenSSH for Windows is a downstream fork of OpenBSD’s OpenSSH‑Portable project, requiring continuous effort to track upstream changes, maintain Windows compatibility, and prevent regressions. This session explores how GitHub Copilot and PowerShell MCP tooling can streamline the maintenance workflow while preserving stability. It focuses on human‑in‑the‑loop engineering patterns for triaging merge conflicts, analyzing build logs, and performing simple validation. The session is designed both for those curious about the realities of open‑source project maintenance and for those looking to leverage GitHub Copilot and MCP to modernize their own workflows.
