# Develop, maintain and distribute PowerShell modules in isolation

## Abstract

GitHub, Azure DevOps and all the other cloud-based dev stacks have become a staple in PowerShell code development many years ago. Publishing a module to the PowerShell Gallery enables seamless distribution to a variety of devices. But what if your organization requires you to work in isolation, or prohibits you from sharing code with third parties altogether? Did I mention I spent a lot of time in regulated industries and government?

Rejoice, because all is not lost. In this session, we will walk through creating a complete collaborative module development, signing, publishing and distribution stack completely within your domain of influence - even on premises, if that's how your environment is built. Parts of it can also be used to create a "poor man's module factory" or a "shadow scripting infrastructure", in case you cannot get Management to buy into a modern stack.
