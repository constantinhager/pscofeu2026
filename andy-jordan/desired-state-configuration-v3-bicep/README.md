# Desired State Configuration v3 & Bicep

## Abstract

Learn how to author DSCv3 configuration documents in Bicep — an infrastructure-as-code programming language designed for describing and deploying resources. While originally created for Azure resources, hear about how we've extended it to include DSCv3 resources and why you should use it (especially if transitioning from legacy DSC PowerShell keyword documents).

When compared to the equivalent JSON or YAML documents, Bicep documents are more concise and easier to read. Since Bicep syntax is declarative, using it doesn't require prior knowledge of programming languages. You just specify which resources and resource properties you want, complete with IntelliSense in VS Code, and gain powerful programming features such as modules and loops. 

Includes a live demo and reusable examples.
